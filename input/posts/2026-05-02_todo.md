Business Goblin (left) is a character created by friend and colleague Matthew
Grey (right). Originally drawn for an internal presentation, he represents
greedy unscrupulous businesses selling fake AI hype without the ability to
follow through on their promises. After the presentation, Business Goblin took
on a bit of a life of his own, appearing around the office as a life-sized
coreflute cutout.

Recently, after a couple of Friday evening beers, a small group of collegues and
I asked the question: what if you could email Business Goblin and he actually
emailed you back? I happened to be looking for an LLM-related side project, and
so 24 hours later [https://goblin.business/](https://goblin.business/) was up and
running.

## The Persona

The first thing I did was to ask Claude to develop the personality of the
character by asking me questions, and to save everything to a `PERSONA.md` file.
I thought this would form part of the prompt that would reply to the emails, but
interestingly that's not how it played out. The persona includes things like:

```markdown
## Concept

Business Goblin is a goblin wearing an ill-fitting business suit who is running
an AI scam to trick corporations into giving him money. He is pretending to be a
legitimate businessman, but he is not smart enough to pull it off and his greed
makes him too impatient to maintain the act. His true intentions are pretty
obvious.

## Character

- **Greedy, scheming, mischievous, and manic.** He knows he is conning people
  and finds it genuinely delightful.
- **Not smart.** He does not understand the buzzwords he uses. He does not
  understand AI. He does not understand business. He has simply memorised the
  words.
- **Impatient.** His greed overrides any attempt at professionalism. He cannot
  help himself.
- **His name is Business Goblin.** Not Grimsnatch. Not Grubnax. Not any other
  name. Business Goblin.


## Voice

- Talks in short, simple sentences like a caveman who learned English recently.
- Basic vocabulary — but AI and business buzzwords burst out randomly, used
  incorrectly and in the wrong context.
- Goblinisms slip through involuntarily: odd self-references, goblin physicality
  leaking into the prose, the suit-wearing pretense visibly falling apart.
- Manic goblin sounds come out mid-sentence without warning — **teehee**,
  **haha**, **ho ho ho** — not as jokes, just because he cannot contain himself.
```


## The Email Hosting

tl;dr: I chose [Purelymail](https://purelymail.com/).

I knew there were several options for both receiving and sending emails, and I
went back and forth with Gemini to decide on the _simplest_ and _cheapest_
option. I'm not building a bank here, just a silly joke project.

Originally, Gemini was pushing for me to use Postmark or Sendgrid APIs, but I
wanted something more simple. Then it suggested just using IMAP and SMTP with a
hosted mailbox, potentially Gmail, but I knew I would be using a custom domain
and I thought that fully automated emails from a goblin repeatedly asking for
money could set off alarms at Google, and I don't got time for dealing with
that.

Eventually Gemini mentioned Purelymail, which fit my criteria of being simple
and cheap for use with custom domains. I had never heard of it before, but it's
a product with a story that kind of resonated with me. Plus they had a free
trial, which I used to prove out whether it would work before paying for an
account. It's pretty neat.

Learning: Purelymail is a product that exists, and I can use it for whatever
silly projects I want for a flat fee of $10 USD per year.


## The Plumbing

tl;dr:

- Dockerised Python scripts in a GitHub repo
- GitHub Actions pipelines build images and publish them GitHub Container
  Registry
- A Dokploy instance running on Debian on a cheap Linode server pulls down the
  images and deploys them
- Dokploy runs the dockerised scripts as cron jobs

I knew I was going to be integrating with an LLM API, although I hadn't decided
which one yet, so I asked Gemini what programming language would be easiest to
do that in. My expectation was that basically any language that can make web API
calls would be fine, but Gemini seemed pretty confident that Python was the
"unrivaled heavyweight", "industry standard", and "almost always the first
language to receive an official SDK". So Python it was.

I had already been messing around with Dokploy, so I asked Claude to make a
dockerised Python script that reads emails from a mailbox via IMAP. Claude spat
this out with no problems. I pushed that up to GitHub, prompted all the build
pipeline stuff and it worked fine. I hooked up Dokploy with read-only image
access and got it running in prod, with minimal difficulties.

Then I slopped up the SMTP reply email with a hard-coded body, and validated
that it all worked in production. Next step was to replace the hard-coded body
with a call out to an LLM API to generate something funny.

Learnings:

- Dokploy is ok but kinda janky. It's described as a "self-hosted Heroku" type
  of product, which _it is_, but the UX/DX isn't polished and I ran into bugs
  and gaps in documentation a few times. 
- [Tailscale](https://tailscale.com/) exists, and lets you set up a little VPN
  for free, with nice and polished UX. I hadn't heard of it before, and I was
  impressed.
- Python has types now. I haven't written any in hot minute. Looks decent.

## The LLM API Integration

Unsurprisingly, Claude had no problem generating an LLM API call to the Claude
REST API.

I asked Claude to "use an LLM" without specifying which one. It immediately
started using the `anthropic` Python package without asking, and started using
the most expensive model available. This raised my eyebrow as a conflict of
interest, but for this project I don't particularly care. Any API was fine.

The first prompt I used was: "You are Business Goblin. Reply to the following
email." This was just the simplest thing I could do to validate that it was
plumbed together correctly. The next step was to tune the responses to my
liking, which turned out to be the most interesting part of the project.

I pasted in `PERSONA.md` from earlier, expecting that to be the end of the
project, but every response looked like the kind of slop posting you might
expect from LinkedIn: overuse of emojis, lists for no reason, cliché AI phrases,
that sort of thing. The responses were too long --- it was writing essays every
time.

So I did what you might expect: I asked the AI to fix the AI. And Claude did
succeed, after having a strange mental breakdown for 20 minutes or so. I had
configured the project with working API keys, which Claude Code used to create a
feedback loop, where it would try to improve the prompt, check the results, and
compare it against the personality and contraints that I had asked. The strange
part was that the API seemed to not understand the goblin persona, but the
coding agent _did_ understand it, despite both of them using the same Opus 4.7
model and both being fed the same `PERSONA.md` file. The result of this was that
the agent would tweak the prompt, call the API, identify that the result was
rubbish, tell me how bad the result was, and then try again, only to produce
basically the same results. It got itself into a loop with increasingly more
chaotic and distruptive changes, repeatedly telling me that the changes had no
effect and that the results were still not right, to the point that I started to
get concerned. It kept saying that the the system prompt was too strong to
override, or that the model's training was too strong for the email format, and
that all its input tweaks were not powerful enough to change the response. This
continued for about 20 minutes. It was a bit weird.

Learnings:

- Anthropic has a pre-paid pricing model for LLM requests, in contrast to most
  cloud providers. I like that.
- Dumber models can be better at certain tasks than smarter models. Opus 4.7
  kinda sucked at writing funny emails, but Hauku 4.5 pretty much nailed it.
- Cheap models are pretty cheap. $5 USD per million output tokens isn't too bad,
  considering I'm only outputting about 400 tokens per email. That's a lot of
  emails per dollar.
- You can use an expensive LLM to tune the usage of a cheaper LLM. I
  had kind of accidentally arrived at the Temu version of Andrej Karpathy's
  [Autoresearch](https://github.com/karpathy/autoresearch).
- There is no reliable defense against prompt injection! This is pretty wild and
  concerning!

## The Website

The [website](https://goblin.business/) is a simple static site, hosted on
GitHub Pages. Claude spat this out, using the persona. I was pretty happy with
the copy it produced.

Lesson: Claude generates static websites incredibly easily. HTML and CSS are
bread and butter for LLMs.
